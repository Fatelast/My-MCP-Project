# 产品需求文档 (PRD)

## 1. 概述

- **功能名称**：日本麻将（立直麻雀）单机游戏
- **版本**：1.0
- **日期**：2026-03-17
- **作者**：PM Agent

---

## 摘要

> 下游 Agent 请优先阅读本节，需要细节时再查阅完整文档。

- **核心目标**：在 Web 浏览器中实现一款可玩性强的日本麻将单机游戏，玩家对战3个AI对手
- **目标用户**：了解或想学习日本麻将的玩家（PC + 移动端浏览器）
- **关键功能**：
  1. 完整立直麻雀规则引擎（基础20+役种、点数计算）
  2. 三种难度AI对手
  3. 响应式界面（PC + 手机）
  4. 牌局状态可视化（手牌、河牌、副露、宝牌指示牌）
  5. 分数结算与排行（本局）
- **技术约束**：React + TypeScript，纯前端单机，无需后端
- **优先级**：MVP 先实现完整对局流程，AI可先用基础策略

## 4. 目标用户

### 用户画像 1：麻将爱好者
- **基本特征**：18-40岁，了解日本麻将基本规则
- **行为特征**：碎片时间游戏，喜欢练习技术
- **核心需求**：规则正确、AI有挑战性
- **痛点场景**：想练习但没有对手

### 用户画像 2：麻将学习者
- **基本特征**：任意年龄，刚接触日本麻将
- **行为特征**：边玩边学，需要提示
- **核心需求**：有操作提示，能看到役种说明
- **痛点场景**：不知道自己能不能和牌

## 5. 功能需求

### FR-001：对局核心流程
- **需求描述**：完整的一局游戏流程（东风战/东南战）
- **优先级**：P0
- **验收标准**：
  - [ ] 4人自动轮流出牌
  - [ ] 玩家可手动选择出哪张牌
  - [ ] 满足条件自动提示吃/碰/杠/和
  - [ ] 流局（摸完所有牌）正确处理
  - [ ] 亲家（庄家）轮转正确

### FR-002：役种判定引擎
- **需求描述**：判断和牌时成立的役种
- **优先级**：P0
- **基础役种列表**：
  - 立直、一发、门清自摸、役牌（各三元/场风/自风）
  - 断幺九、平和、一盃口、混全带幺九
  - 一气通贯、三色同顺、七对子、对对和
  - 混一色、清一色、小三元、混老头
  - 海底摸月、河底捞鱼、嶺上开花、抢杠
- **验收标准**：
  - [ ] 上述役种均可正确判定
  - [ ] 复合役种可同时成立
  - [ ] 满贯/跳满/倍满/三倍满/役满正确计算

### FR-003：点数计算
- **需求描述**：和牌后计算符数、番数、点数
- **优先级**：P0
- **验收标准**：
  - [ ] 基本符数计算正确（门符/雀头符/面子符/和牌符）
  - [ ] 亲家/子家点数差异正确

[TM_CONTINUE_START:tmc-1gy0cqm]
本场和牌/支付点数正确
  - [ ] 本场棒（连庄）点数正确
  - [ ] 立直棒正确处理

### FR-004：AI 对手
- **需求描述**：3个AI对手可自动完成摸牌/出牌/鸣牌决策
- **优先级**：P0
- **验收标准**：
  - [ ] AI可正确判断是否鸣牌（吃/碰/杠）
  - [ ] AI可正确判断是否和牌
  - [ ] AI出牌策略合理（不出危险牌优先）
  - [ ] AI有基础难度差异

### FR-005：响应式界面
- **需求描述**：游戏界面在PC和手机上均可正常游玩
- **优先级**：P0
- **验收标准**：
  - [ ] 桌面端（1024px+）显示完整牌桌
  - [ ] 手机端（375px+）可正常操作
  - [ ] 触控操作可选牌/出牌

### FR-006：操作提示
- **需求描述**：在合法操作时显示操作按钮
- **优先级**：P1
- **验收标准**：
  - [ ] 可以吃牌时显示"吃"按钮及组合选择
  - [ ] 可以碰牌时显示"碰"按钮
  - [ ] 可以杠牌时显示"杠"按钮
  - [ ] 可以和牌时显示"荣"或"自摸"按钮
  - [ ] 超时自动跳过

### FR-007：和牌结果展示
- **需求描述**：和牌后展示役种、番数、点数
- **优先级**：P1
- **验收标准**：
  - [ ] 显示所有成立役种名称
  - [ ] 显示符数+番数
  - [ ] 显示支付点数明细
  - [ ] 展示完整手牌

## 6. 非功能需求

### NFR-001：性能
- 游戏逻辑响应 < 100ms
- AI出牌延迟模拟 0.5-1.5s（拟人感）
- 首屏加载 < 3s

### NFR-002：兼容性
- Chrome 90+ / Safari 14+ / Firefox 88+
- iOS Safari 14+ / Android Chrome 90+

### NFR-003：可维护性
- TypeScript 严格模式
- 游戏逻辑与UI完全解耦
- 役种引擎可独立测试

## 7. 范围定义

### 本期范围（In Scope）
- 完整单局对战流程
- 基础20+役种判定
- 点数计算（含符数）
- 基础AI策略
- PC+手机响应式
- Git初始化

### 范围外（Out of Scope）
- 在线多人对战
- 用户账号/登录
- 段位/天梯系统
- 役满（国士无双等）暂不包含
- 声音（可后续迭代）
